# メモリー専用リポジトリ セットアップ手順

予約システムと完全分離した「写真メモリー専用」システムの構築手順です。

---

## 構成（完成後）

```
GitHub: masa360/line-memory（新規作成）
  ├── gas/
  │   ├── Code.gs        ← GAS本体（画像保存・履歴API・LIFF画面配信）
  │   └── MemoryPage.html← LIFF画面（HTML/CSS/JS）
  └── README.md

Vercel: line-memory（新規プロジェクト）
  ← 不要（GAS単体で完結するため）

GAS: memory-gas（新規プロジェクト）
  ← line-memory リポジトリの gas/ フォルダ内のコードを貼り付け

LINE: もりもり用公式アカウント
  ← メモリー専用LIFFを追加
```

---

## Step 1: GitHub 新規リポジトリを作る

1. [GitHub](https://github.com/new) にアクセス
2. Repository name: `line-memory`
3. Visibility: Private（推奨）
4. `Create repository`

---

## Step 2: ローカルにリポジトリを作る

Cursor のターミナルで以下を実行：

```bash
cd d:\cursur
mkdir line-memory
cd line-memory
git init
git remote add origin https://github.com/masa360/line-memory.git
```

---

## Step 3: ファイルを用意する

以下の2ファイルを `gas/` フォルダに作成します。

**`gas/Code.gs`**
- `d:\cursur\reserve-LIFF\` の `memory-standalone\gas\Code.gs`（削除前のもの）を使う
- または下の「必要な関数一覧」を参照して新規作成

**`gas/MemoryPage.html`**
- `d:\cursur\reserve-LIFF\memory-standalone\gas\MemoryPage.html` を使う

> コードの中身はこのCursorセッションで「新規作成して」と依頼すれば、すぐに出力します。

---

## Step 4: GAS 新規プロジェクトを作る

1. [script.google.com](https://script.google.com) → 「新しいプロジェクト」
2. プロジェクト名: `memory-gas`
3. `コード.gs` に `gas/Code.gs` の内容を貼り付ける
4. `+` ボタン → HTMLファイルを追加 → ファイル名 `MemoryPage`
5. `MemoryPage.html` の内容を貼り付ける

---

## Step 5: スクリプトプロパティを設定する

GAS「プロジェクトの設定」→「スクリプトのプロパティ」に追加：

| キー | 値 |
| :--- | :--- |
| `MEMORY_LINE_ACCESS_TOKEN` | もりもり用LINE公式のアクセストークン |
| `MEMORY_DRIVE_FOLDER_ID` | 写真保存先DriveフォルダのID |
| `MEMORY_SPREADSHEET_ID` | 記録用スプレッドシートのID |
| `MEMORY_LIFF_ID` | 作成したメモリーLIFF ID（Step 7で取得） |

---

## Step 6: GAS を初期化してデプロイする

1. GAS エディタで `setupMemorySheet_` を手動実行（初回1回だけ）
   - スプレッドシートに `PhotoLog` シートが作られる
2. 「デプロイ」→「新しいデプロイ」
   - 種類: ウェブアプリ
   - 次のユーザーとして実行: 自分
   - アクセスできるユーザー: 全員
3. `/exec` URL を控える（Webhook URLとしても使う）

---

## Step 7: LINE Developers でメモリー専用 LIFF を作る

1. [LINE Developers](https://developers.line.biz/) → もりもり用チャネル → `LIFF` タブ
2. 「追加」ボタン
3. 設定値：

| 項目 | 値 |
| :--- | :--- |
| LIFFアプリ名 | `メモリー` |
| サイズ | `Full` |
| エンドポイントURL | GAS の `/exec` URL |
| スコープ | `profile` にチェック |

4. 発行された **LIFF ID** を `MEMORY_LIFF_ID` としてスクリプトプロパティに設定する

---

## Step 8: LINE Webhook を設定する

LINE Developers → Messaging API タブ：

- Webhook URL: GAS の `/exec` URL
- Webhook の利用: ON
- 「検証」で 200 OK を確認

---

## Step 9: リッチメニューに「メモリー」ボタンを設定する

- ボタンのURL: `https://liff.line.me/{MEMORY_LIFF_ID}`

---

## Step 10: 動作テスト

1. LINEのトーク画面で写真を送信
2. Driveに写真が保存されることを確認
3. スプレッドシートの `PhotoLog` に行が追加されることを確認
4. リッチメニュー「メモリー」をタップ → LIFF が開いて写真が表示される

---

## 運用メモ

- **予約システムとは完全独立**（コードもGASもVercelも別）
- 店舗へ渡すときは Drive / Spreadsheet / GAS への権限を付与するだけでよい
- もりもり用LINE公式アカウントのみで使用（予約-only公式では使わない）
- GASのコードはこのリポジトリ（`line-memory`）で管理する

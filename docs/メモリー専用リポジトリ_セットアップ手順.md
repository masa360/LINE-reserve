# メモリー（Vercel）セットアップ手順

**更新（2026年3月）**  
以前の「`line-memory` 専用リポジトリ＋GAS分割」案は **採用しません**。**GitHub は `LINE-reserve` のみ**、**GAS・スプレッドシートも予約と同じ 1 本**のまま、**表示だけ** `memory-viewer`（Vercel）に載せます。

---

## 構成（完成後）

```
GitHub: masa360/LINE-reserve（本命・これだけ）
  ├── （予約用 Next.js）
  └── memory-viewer/          ← メモリー用 Next.js（薄い表示専用）

GAS: 予約と同じデプロイ 1 本
  - Webhook・画像保存・MemberPhotoLog・GET ?action=photos はそのまま

Vercel: プロジェクトを 2 つ（どちらも同じ LINE-reserve を Import）
  1. 予約用 … Root Directory = リポジトリルート（または既存どおり）
  2. メモリー用 … Root Directory = memory-viewer

LINE: メモリー専用 LIFF
  - エンドポイント URL = メモリー用 Vercel の https://xxxx.vercel.app/
```

---

## Step 1: Vercel でメモリー用プロジェクトを追加

1. Vercel → **Add New…** → **Project**
2. **Import** するリポジトリ: **`LINE-reserve`**
3. 設定:
   - **Root Directory**: **`memory-viewer`**（Edit で指定）
   - Framework: Next.js（自動検出でよい）
4. **Environment Variables**（メモリー用プロジェクトだけに設定）:

   | 名前 | 説明 |
   | :--- | :--- |
   | `GAS_WEBAPP_URL` | 予約でも使っている **同じ** GAS `/exec` の URL |
   | `NEXT_PUBLIC_LIFF_ID` | メモリー用 LIFF の ID（Step 2 で発行） |

5. **Deploy**

---

## Step 2: LINE Developers（メモリー専用 LIFF）

1. [LINE Developers](https://developers.line.biz/) → 該当チャネル → **LIFF**
2. **追加**
3. 推奨設定:

   | 項目 | 値 |
   | :--- | :--- |
   | LIFF アプリ名 | `メモリー` など |
   | サイズ | Full |
   | エンドポイント URL | `https://（メモリー用 Vercel のドメイン）/` **※ GAS の URL ではない** |
   | スコープ | `profile` |

4. 発行された **LIFF ID** を Vercel の `NEXT_PUBLIC_LIFF_ID` に反映し、**Redeploy**（必要なら）

---

## Step 3: リッチメニュー

- 「メモリー」ボタンの URL: `https://liff.line.me/{メモリー用LIFFのID}`

---

## Step 4: 動作確認

1. LINE で画像送信 → Drive・`MemberPhotoLog` に従来どおり記録される
2. リッチメニュー「メモリー」→ Vercel の画面で一覧が表示される

---

## 運用メモ

- **`line-memory` リポジトリは不要**（削除・未使用で問題ありません）
- ソースの正は **`LINE-reserve`** のみに集約する
- GAS の `doGet` で `action` なしのときに返す `MemoryPage.html` は、LIFF を Vercel に切り替えた後は **実際には使われません**（残しておいても可）

## ローカル開発（任意）

```bash
cd memory-viewer
cp .env.example .env.local
# GAS_WEBAPP_URL / NEXT_PUBLIC_LIFF_ID を記入
npm install
npm run dev
```

詳細はリポジトリ内 `memory-viewer/README.md` を参照してください。
